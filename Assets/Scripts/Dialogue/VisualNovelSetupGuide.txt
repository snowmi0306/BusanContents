Visual Novel StagePortal NPC Setup Guide

Important project rule:
- StagePortal is the stage-end NPC in this project.
- Do not create a separate Portal object.
- Do not create a separate NPC_StageGoal object.
- Do not create another goal trigger script.
- Do not deactivate StagePortal at scene start; it must stay active as the NPC.

Hierarchy example:
Stage
- _SYSTEM
  - EventSystem
  - DialogueManager (active)
    - VisualNovelDialogueController
- _UI
  - Canvas
  - VisualNovelCanvas

  - Player

VisualNovelDialogueController:
- Attach this component to the always-active DialogueManager object.
- Do not attach this component to the initially inactive VisualNovelPanel.
- Assign Dialogue Panel to VisualNovelPanel.
- Assign Dandi Portrait, NPC Portrait, Speaker Name Text, Dialogue Text, and Next Button.
- Leave the NextButton Inspector OnClick list empty; VisualNovelDialogueController registers the click handler in code.
- Keep Next Key as Space and Characters Per Second around 35.
- Enable Show Cursor During Dialogue so Cursor.visible becomes true and Cursor.lockState becomes None during dialogue.
- Leave Restore Cursor After Dialogue disabled unless a scene specifically needs the old cursor state restored.

StagePortal / StageGoal:
- Use the existing StageGoal component on StagePortal.
- StageGoal is both the NPC dialogue trigger and the stage clear trigger.
- Set Use Dialogue Before Clear to true.
- Assign the scene's DialogueManager VisualNovelDialogueController, or leave it empty so StageGoal can find it at runtime.
- Fill Dialogue Lines with NPC and Dandi lines.
- Assign Dandi Portrait Sprite and the current stage NPC Portrait Sprite.
- Set Require Interact Key to false for automatic dialogue when the player touches StagePortal.
- If Require Interact Key is true, assign Interact Hint and use Interact Key, usually E.
- Assign Transition Panel if a fade/cover panel should appear after dialogue.
- Set Load Delay After Dialogue to about 0.5 seconds.

Stage-specific StagePortal values:
Stage1:
- SpriteRenderer = ClearVisual_NPC1
- Dandi Portrait Sprite = ClearVisual_Dandi
- NPC Portrait Sprite = ClearVisual_NPC1
- Next Scene Name = Stage2

Stage2:

Stage3:

- Dandi Portrait Sprite = ClearVisual_Dandi
- NPC Portrait Sprite = ClearVisual_NPC3
- Next Scene Name = ClearScene

Dialogue behavior checklist:
- Player touches StagePortal Trigger.
- StageGoal calls VisualNovelDialogueController.StartDialogue().
- Dialogue panel opens and both portraits are shown if their Image references and Sprites are assigned.
- The active speaker portrait alpha is 1.0; the inactive speaker portrait alpha is 0.5.
- Text types one character at a time.
- Space and NextButton do the same thing.
- Pressing Space or NextButton while typing instantly completes the current line.
- Pressing Space or NextButton after the line is complete advances to the next line.
- After the last line, StageGoal shows transitionPanel, waits loadDelayAfterDialogue, then loads nextSceneName.
